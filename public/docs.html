Frontend → Backend → Database

HTML Form → fetch() → Express Route → Parameterized SQL → PostgreSQL

The frontend never connects directly to PostgreSQL. It sends HTTP requests to Express, and Express safely talks to the database.

REST API to SQL Mapping

POST maps to INSERT for creating rows.

GET maps to SELECT for reading rows.

PATCH and PUT map to UPDATE for changing rows.

DELETE maps to DELETE for removing rows.

DML Meaning

DML means Data Manipulation Language. The main DML commands are INSERT, SELECT, UPDATE, and DELETE.

Primary Key vs Foreign Key

A primary key uniquely identifies one row in its own table. A foreign key points to a primary key in another table and creates a relationship.

UUID

A UUID is a 128-bit identifier. This project uses gen_random_uuid() from PostgreSQL pgcrypto to generate IDs.

UUIDs help distributed systems because different services or machines can create IDs without asking one central counter for the next number.

Documents and File Storage

PostgreSQL can store binary data, but real projects usually store files in object storage or a file service, then store only the file URL in the database.

SQL Injection

Unsafe example: "SELECT * FROM students WHERE email = '" + email + "'"

Safe example: SELECT * FROM students WHERE email = $1 with values passed separately as [email].

Constraints

NOT NULL requires a value. UNIQUE prevents duplicates. CHECK enforces a rule, like positive course duration or valid application status.

Referential Integrity

Foreign keys prevent applications from referencing students or courses that do not exist. They keep relationships valid.

Migrations

An up script creates or changes schema objects. A down script reverses the change, such as dropping a table. This makes schema changes repeatable and reviewable.
